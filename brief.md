Brief and Instructions.
We are creating a tool that helps create edit and organize documents and diagrams covering all steps of software design. Documents such as project plan, test plan, requirements, user stories etc should have a fixed format when possible forcing the user into structure. E.g. user stories document has the format As a [type of user], I want to [perform an action] so that I can [achieve a goal/benefit]" where the user fill in the blanks. And requirements is a table with fixed cells and labels, however it is usually structured. Diagrams should be free form. All documents should be exportable to a format tailored to LLM coding agents. designdoc.md describes our tech stack among other things but lacks some things and is badly formatted. Read it and edit to make it more comprehensive and implementation ready. Development Roadmap should be for AI agents.
